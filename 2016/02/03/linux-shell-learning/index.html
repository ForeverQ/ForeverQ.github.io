Linux Shell 中 >/dev/null 2>&1 的含义

发表于 2016-02-03   |   分类于 学习

今天研究 V2EX 签到脚本的时候，看到 Crontab 脚本中 >/dev/null 2>&1 这样的写法甚是疑惑，于是查阅资料一探究竟。

首先解释一下此命令的含义是「将程序的标准输出和错误信息一同丢弃」。

具体来说，

/dev/null 代表空设备文件，即不存在的设备；

> 代表重定向，比如 echo test>1.txt 就是把 text 这个文本输出到 1.txt 这个文件中；

关于 1 和 2，在 /usr/include/unistd.h 中，可以找到如下代码：

#define STDIN_FILENO 0 #define STDOUT_FILENO 1 #define STDERR_FILENO 2

这是三种不同的流，于是 1 代表 stdout，2 代表 stderr；

&> 意为「等同于」，表示左边的内容以符号右边的形式输出，所以 2&>1 就是把 stderr 作为 stdout 输出。

总的来说，>/dev/null 2>&1 首先通过 2&>1 定义了把 stderr 作为 stdout 流输出，然后再把 stdout 的内容写入 /dev/null，也就是丢弃掉。当然，如果把 /dev/null 改成别的，即可记录控制台日志了。
